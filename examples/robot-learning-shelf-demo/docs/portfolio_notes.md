# 作品集整理说明（Chapter 20）

本文件用于把 `robot-learning-shelf-demo` 从“学习项目”收束为“可展示的作品集”。

## 1. 作品集目标

面向以下使用场景：

- GitHub 项目主页展示；
- 简历中的项目链接；
- 面试中的系统讲解；
- 技术博客 / 分享材料沉淀。

## 2. 推荐展示结构

1. **项目概述**：说明项目目标——围绕桌面 pick-and-place，建立一个从 episode、数据集、训练、评测到失败闭环的机器人学习教学主线；
2. **系统架构图**：展示 perception / policy / evaluation / failure loop；
3. **关键结果**：给出 `policy_v1`、`policy_v2` 的 success rate 与失败原因变化；
4. **数据说明**：说明 `dataset_v1_scripted`、`dataset_teleop_demo`、`dataset_v0` 的关系；
5. **演示视频建议**：至少准备“成功案例 + 失败案例 + 失败分析 + 改进后结果”四段视频；
6. **扩展方向**：从桌面任务扩展到理货机器人、货架异常检测、商品扶正和移动操作。

## 3. GitHub README 的最小结构

- 项目简介
- 为什么做这个项目
- 项目结构
- 运行顺序
- 主要实验结果
- 数据与评测说明
- 失败分析与改进
- 后续路线图

## 4. 展示重点

### 4.1 不只是展示“成功视频”

优秀的机器人作品集，重点不是“炫一个成功 demo”，而是展示：

- 你如何定义任务；
- 你如何设计数据结构；
- 你如何做数据采集与质检；
- 你如何建立评测协议；
- 你如何分析失败并推动下一轮迭代。

### 4.2 强调工程闭环

建议在介绍中明确强调以下主线：

```text
任务定义 -> episode 结构 -> 数据采集 -> 模型训练 -> 评测 -> 失败分析 -> 数据回流 -> 再训练
```

### 4.3 强调“从桌面到货架”的扩展能力

如果你希望把项目和真实职业机会连接起来，应该突出：

- 从桌面操作任务到货架理货任务的迁移思路；
- 感知层与操作层的分工；
- 自动驾驶背景如何迁移到机器人感知、空间理解和数据闭环。

## 5. 可直接引用的成果亮点

- 构建了从 synthetic episode 到 scripted dataset、teleop dataset、dataset_v0 的完整教学数据链路；
- 训练了一个 ACT-like baseline，并建立了离线训练与结构化评测脚本；
- 建立了基于 CSV 的评测协议、failure taxonomy 和补采数据计划；
- 将桌面 pick-and-place 扩展到了货架异常检测与理货机器人方向。

## 6. 面试中建议如何讲

推荐遵循下面的 4 分钟结构：

1. **一句话介绍项目**；
2. **讲主线闭环**；
3. **讲最有代表性的失败分析案例**；
4. **讲下一步如何扩展到真实场景**。
