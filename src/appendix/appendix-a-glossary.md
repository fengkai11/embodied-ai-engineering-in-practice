# 附录 A：全书术语表

本附录用于统一全书核心术语。它不是百科词典，而是服务于本书主线项目的“工作术语表”：读者在阅读章节、运行代码、整理项目时，可以快速查到关键概念在本书中的含义。

---

## A.1 具身智能与机器人学习

### Embodied AI / 具身智能

具身智能强调智能体并不是只在文本或屏幕中推理，而是通过身体、传感器、执行器与物理世界交互。它关心视觉、语言、动作、空间、接触和反馈之间的闭环。

在本书中，具身智能不是简单的“机器人 + LLM”，而是一套围绕任务、数据、策略、评测和失败回流的工程系统。

### Robot Learning / 机器人学习

机器人学习指通过数据、示范、试错或模型训练，让机器人获得感知、决策和操作能力。它不同于只写规则控制，也不同于只训练视觉模型，而是把观测、动作和环境反馈串成学习闭环。

### Mobile Manipulation / 移动操作

移动操作指“移动底盘 + 机械臂 + 末端执行器”的组合系统。第 19 章的货架理货机器人就是典型移动操作场景。

## A.2 数据与策略基础术语

### Observation / 观测

机器人在某一时刻从传感器、状态估计和任务上下文中获得的信息。例如 RGB 图像、深度图、机器人关节状态、末端位姿、语言指令等。

### State / 状态

状态是对环境和机器人当前情况的结构化描述。它可以来自真实传感器，也可以是仿真或日志中的可解释字段。例如 `ee_pose_xyzrpy`、`gripper_open`、`object_pose_xyz`。

### Action / 动作

动作是策略输出并发送给机器人控制系统的命令。本书主线项目中常用的动作形式是末端执行器位姿增量和夹爪开合量。

### Policy / 策略

策略是从 observation / state 到 action 的映射。它可以是规则脚本、行为克隆模型、ACT-like baseline、Diffusion Policy 或 VLA 模型。

### Episode / 轨迹片段

episode 是一次完整任务或任务片段的记录，通常包含图像、状态、动作、元数据和成功/失败标签。本书所有数据集都围绕 episode 组织。

### Rollout / 策略执行

rollout 指让策略在环境中连续执行，观察其是否能完成任务。离线 loss 只能说明拟合情况，rollout 才更接近真实任务表现。

## A.3 模仿学习与策略模型

### Behavior Cloning / 行为克隆

行为克隆是一种最基础的模仿学习方法：给定专家示范数据，训练模型从观测预测动作。它简单、易实现，但容易在长期执行中产生误差累积。

### Imitation Learning / 模仿学习

模仿学习是一类从专家示范中学习策略的方法。示范可以来自人类遥操作，也可以来自 scripted expert 或其他自动专家。

### ACT / Action Chunking Transformer

ACT 的核心思想是一次预测一段未来动作，即 action chunk。它适合很多局部连续操作任务，尤其适合低成本机械臂学习项目作为强基线。

### Action Chunk / 动作块

动作块是一段连续动作序列。例如当前时刻输入一个 observation，模型输出未来 4 步或 8 步动作。

### Diffusion Policy / 扩散策略

Diffusion Policy 使用扩散建模思想生成动作序列，适合表达复杂、多模态动作分布。本书将其作为重要进阶方向，而不是第一版 baseline。

### VLA / Vision-Language-Action

VLA 模型把视觉、语言和动作连接起来，目标是从视觉场景与语言指令直接生成机器人可执行动作。

### VLM / Vision-Language Model

VLM 主要解决视觉和语言理解问题，例如看图回答问题、图文匹配、场景描述等。VLA 相比 VLM 进一步引入动作输出和具身约束。

## A.4 数据采集与数据集术语

### Scripted Expert / 规则专家

用人工规则或状态机写出的 expert policy。它不一定泛化强，但适合做系统联调、生成起步数据和验证任务链路。

### Teleoperation / 遥操作

人类通过键盘、手柄、VR、主从机械臂等设备控制机器人完成任务，并把过程记录成 episode。

### Dataset / 数据集

由多条 episode 组成，并带有版本、划分、统计和数据卡片的数据资产。数据集不是 episode 的简单堆叠，而是有组织的学习分布。

### dataset_v0

本书第一版可训练数据集，由 scripted 数据和 teleop 数据整合而来，用于训练 ACT-like baseline。

### dataset_v1

指在失败分析之后，通过 targeted data collection 补采得到的下一版数据集。它强调“针对失败补数据”，不是盲目堆数据。

## A.5 机器人系统术语

### ROS2

机器人系统中常用的通信和中间件框架。本书使用 ROS2 的概念帮助读者理解 node、topic、message 与 recorder 的关系。

### rosbag

ROS 系统中的数据录制格式。它可以保存 topic 数据，后续再回放或转换为训练用 episode。

### TF

TF 用于维护机器人系统中的坐标系变换关系，例如 `base_link`、`camera_link`、`end_effector` 之间的关系。

### base_link

机器人基座坐标系。许多机械臂目标位姿最终都要转换到这个参考系下表达。

### camera_link

相机坐标系。视觉检测结果通常先出现在相机或图像坐标系中。

### end_effector

末端执行器坐标系，通常对应夹爪或工具中心点。

### MoveIt2

常用的机械臂运动规划框架。它负责 IK、轨迹规划、碰撞检测等操作任务中的关键能力。

## A.6 评测与失败闭环术语

### Success Rate / 成功率

策略在固定评测协议下完成任务的比例。成功率必须基于多轮评测，而不能由单次视频代表。

### Completion Time / 完成时间

任务从开始到结束所需时间，用于衡量效率。

### Collision Rate / 碰撞率

执行过程中发生碰撞或碰撞风险的比例。

### Drop Rate / 掉落率

抓取或转运过程中物体掉落的比例。

### Intervention Rate / 干预率

人类或安全系统在执行过程中介入的比例。

### Failure Reason / 失败原因

结构化记录失败类型，例如 `grasp_offset`、`perception_miss`、`collision_risk`、`drop_during_transfer`。

### Failure Taxonomy / 失败分类体系

把失败按感知、策略、控制、环境、数据等层级分类，用于分析问题和指导补采数据。

### Data Flywheel / 数据飞轮

机器人学习中的持续迭代闭环：评测失败 -> 失败分析 -> 补采数据 -> 再训练 -> 再评测。
