# 附录 F：从教学 Demo 到真机系统的差距清单

本附录用于提醒读者：本书主线项目已经完成了完整教学闭环，但它不等于真实商用机器人系统。真实系统会引入大量硬件、感知、控制、安全和部署问题。

---

## F.1 硬件差距

教学 demo 中很多物理细节被简化，而真机会遇到：

- 机械臂关节限制；
- 电机响应延迟；
- 夹爪夹持力不足；
- 工具中心点误差；
- 线缆干扰；
- 急停与安全 IO；
- 长时间运行发热和磨损。

检查清单：

```text
[ ] 机械臂型号与自由度确认
[ ] 末端执行器参数确认
[ ] TCP 标定
[ ] 夹爪开合范围确认
[ ] 急停开关测试
[ ] 电源与通信稳定性检查
```

---

## F.2 感知差距

教学数据常常是干净图像，而真实环境中会遇到：

- 深度噪声；
- 光照变化；
- 反光；
- 遮挡；
- 运动模糊；
- 多相机时间同步；
- 相机外参漂移。

检查清单：

```text
[ ] 相机内参标定
[ ] 相机外参标定
[ ] 深度尺度确认
[ ] 多相机同步检查
[ ] 典型光照条件测试
[ ] 遮挡场景测试
```

---

## F.3 控制差距

教学项目中的 mock control 不代表真实执行：

- IK 可能无解；
- 轨迹可能碰撞；
- 末端位姿可能不可达；
- 夹爪闭合时机可能不准；
- 机械臂执行误差会积累；
- 控制频率和策略频率可能不一致。

检查清单：

```text
[ ] IK 成功率统计
[ ] 轨迹碰撞检测
[ ] 工作空间边界确认
[ ] 控制频率确认
[ ] 夹爪时序测试
[ ] 执行误差记录
```

---

## F.4 数据差距

真实 teleop 数据比教学数据复杂得多：

- 操作员质量不同；
- 示范轨迹有噪声；
- 时间戳对齐困难；
- 失败标签不一致；
- 数据量不足；
- 数据版本混乱。

检查清单：

```text
[ ] episode 格式固定
[ ] 成功/失败标签规范
[ ] failure_reason 枚举固定
[ ] operator_id 记录
[ ] 数据集版本号记录
[ ] 数据卡片生成
```

---

## F.5 评测差距

真实机器人评测不能只看单次视频：

- 必须重复 trials；
- 必须固定协议；
- 必须记录失败原因；
- 必须区分训练场景与泛化场景；
- 必须保留原始日志。

检查清单：

```text
[ ] 固定位置评测
[ ] 随机位置评测
[ ] 光照变化评测
[ ] 物体变化评测
[ ] failure_reason 统计
[ ] 实验报告输出
```

---

## F.6 部署差距

从 notebook / script 到真实部署还需要：

- 实时推理；
- 模型压缩；
- ROS2 节点化；
- 日志系统；
- 监控与报警；
- 安全边界；
- 远程维护。

检查清单：

```text
[ ] 策略推理延迟测量
[ ] ROS2 节点部署
[ ] 日志落盘
[ ] 异常处理
[ ] 人工接管机制
[ ] 版本回滚机制
```

---

## F.7 本书项目如何继续走向真机

推荐路线：

1. 把 mock robot arm 替换为仿真机械臂；
2. 接入真实或仿真的相机；
3. 接入 MoveIt2；
4. 做真实 teleop；
5. 重新构建 dataset_v1；
6. 训练更强 baseline；
7. 做真实评测和失败分析。
