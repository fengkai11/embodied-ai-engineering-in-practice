# 附录 C：VLA 与 Robot Foundation Model 前沿路线图

本附录用于把本书主线与 VLA / Robot Foundation Model 的前沿方向连接起来。它不是 SOTA 复现教程，而是帮助读者理解：为什么本书从任务定义、episode、teleop、ACT、评测和失败闭环讲起，正好是在打 VLA 所需的工程底座。

---

## C.1 VLA 的基本定义

VLA 是 Vision-Language-Action 的缩写。它试图把三类信息连接起来：

- **Vision**：机器人看到什么；
- **Language**：任务如何被语言描述；
- **Action**：机器人应该如何执行。

相比普通 VLM，VLA 的关键难点在于动作输出必须落到具体 embodiment 上：机械臂、夹爪、移动底盘、传感器、控制频率、动作空间都会影响模型设计。

---

## C.2 从 VLM 到 VLA

VLM 解决的是“看懂图像与语言”的问题，例如：

- 看图描述；
- 图文问答；
- 场景理解；
- 目标定位。

VLA 进一步要求模型输出机器人动作，因此会遇到额外问题：

1. 动作如何表示；
2. 动作如何与机器人硬件对应；
3. 数据如何采集；
4. 训练样本如何组织；
5. rollout 如何评测；
6. 失败如何回收。

这些问题正是本书主线项目反复训练的能力。

---

## C.3 代表性路线

### RT 系列

RT 系列代表了将视觉、语言和动作结合，用大规模机器人数据训练通用策略的路线。

### OpenVLA

OpenVLA 类路线代表了开放式 VLA 模型探索，强调将通用视觉语言能力迁移到机器人动作输出。

### π0 / pi-zero 类路线

这类路线关注通用机器人策略、动作生成和多任务操作能力，常与流匹配、动作专家等思想相关。

### Open X-Embodiment

Open X-Embodiment 类数据集强调跨机器人、跨任务数据组织，是通用机器人策略训练的重要数据基础。

### Gemini Robotics / GR00T

这类工业前沿路线强调机器人基础模型、仿真、合成数据、后训练与数据飞轮。

### LeRobot

LeRobot 更适合本书读者实践，因为它强调低成本机器人学习、数据集、ACT 等可落地工具链。

## C.5 后续深入路线

读者可以按以下路线把本书项目继续向 VLA 靠近：

1. 从结构化状态输入扩展到多视角图像输入；
2. 从单一任务扩展到多任务数据集；
3. 从 ACT-like baseline 迁移到 LeRobot ACT；
4. 加入语言指令 embedding；
5. 引入更真实的 teleop 数据；
6. 设计跨场景 evaluation；
7. 分析失败并持续扩展 dataset_v1、dataset_v2。

---

## C.6 学习建议

不要把 VLA 误解为“只要接一个大模型就能控制机器人”。真正难的是：

- 数据；
- 动作空间；
- embodiment grounding；
- 评测；
- 失败闭环；
- 安全与部署。

本书正文的 1–20 章，正是围绕这些底层能力展开。
